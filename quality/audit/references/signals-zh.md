# 中文叙事文本的 AI 信号清单

> 配合 `SKILL.md` 的六维度评估，逐维度查阅。每个维度给"AI 倾向"与"人类倾向"两栏。

## 维度 1 · 句法与节奏

| AI 倾向 | 人类倾向 |
|---|---|
| 句长均匀、节奏平稳（CV < 0.45） | 长短句交错，CV > 0.7 |
| 爱用结构对称、四平八稳的句子 | 敢用极短断句（"啪。""她还在。"）、不完整句 |
| 段段齐整，缺少呼吸感 | 段落有长短差，刻意留气口 |

**典型 AI 句**："那一刻，他的心中涌起一种难以言喻的复杂情绪，眼眸微微低垂，似乎在回忆着什么。"
**典型人句**："他愣了。半天，问一句：你刚才说什么？"

## 维度 2 · 措辞与陈词

| AI 倾向 | 人类倾向 |
|---|---|
| "那一刻/仿佛/一抹/嘴角/眼眸"高密度 | 用具体名词、动词替代 |
| 多类别套路同时偏高（4 类+ 同时爆表） | 偶尔 1 类偏高是类型文特征，不算 AI 铁证 |
| 同一措辞反复 8 字级以上 | 措辞有个人偏好和怪癖 |
| 万金油形容词（"繁华的街道""美味的食物"） | 专有名词、数字、方言等带毛刺的具体性 |

## 维度 3 · 情感与心理

| AI 倾向 | 人类倾向 |
|---|---|
| 直接命名情绪（"复杂的情绪""百感交集""五味杂陈"） | 主观体验改变注意、误判、选择或关系理解 |
| 先假装"展示"再立刻"点明" | 潜台词留白，让读者自己拼 |
| 人物自我剖析得不像真人 | 允许含糊、矛盾、未解决 |
| 情感节拍收束得干净圆满 | 留未解的尾巴 |

**典型 AI 句**："她强忍着泪水，故作坚强地转身离开，心中却已是百感交集。"
**典型人句**："她已经走到门口，又把伞放了回来。雨还在下，她没解释。"

## 维度 4 · 细节与质感

| AI 倾向 | 人类倾向 |
|---|---|
| 细节笼统、可互换 | 细节奇特、私人、有生活质感 |
| 感官描写像清单（视听嗅味触各一句） | 细节服务于"只属于这个场景"的真实 |
| 专有名词、数字、牌子、方言稀少 | 有具体牌子、具体金额、具体尴尬 |

**典型 AI 句**："街道两旁是繁华的店铺，空气中弥漫着诱人的香味。"
**典型人句**："煎饼摊前的队伍排了七八个人，老板娘的铲子在铁板上咣咣响。"

## 维度 5 · 比喻与意象

| AI 倾向 | 人类倾向 |
|---|---|
| 比喻技术上成立但像零件拼装 | 比喻从角色和情境里长出来 |
| 意象陈词（"月光如水""时间凝固""命运齿轮"） | 偶尔笨拙却新鲜的意象 |
| 段尾爱"金句化"升华 | 不强行升华，敢用不押韵的 |
| 章末用"金句 + 问号"当钩子 | 问号来自具体线索、人物选择或现场危机 |
| 同一中心隐喻在文章里被工整对照使用 | 隐喻自然散落，不强求对称 |

## 维度 6 · 声音、连贯与风险

| AI 倾向 | 人类倾向 |
|---|---|
| 声音平滑、得体，像"优等生作文" | 有鲜明立场、口头禅、偏见、幽默 |
| 不犯错、不冒险、不留脏尾巴 | 敢写难看、危险、留尾巴 |
| 长文出现前后矛盾（类幻觉） | 就算有 bug 也是"人式的不一致" |
| 主题闭环过整洁、对仗过工整 | 允许不圆满、允许复杂未解 |

## 制度戏 / 排队戏专项

| AI 倾向 | 人类倾向 |
|---|---|
| 摆桌子、排队、登记、立规矩写成整齐流程 | 有人写、有人犹豫、有人骂、有人问孩子哭不哭 |
| 多人一人一句，每轮精准推进计划 | 只有真正有目标的人说；旁人可沉默、偏题或根本没听清 |
| 条款一次成型，像打印稿 | 漏一条、被打断、划掉重写、补丁字挤进行缝 |
| 每句都推主线，零闲笔 | 两三句真实闲笔：临期货、胶带、退烧贴、破袋子 |
| 微动作均匀撒点，人物做完仍按原计划走 | 情绪改变注意、判断、选择或关系位置；无后果动作直接删 |
| 修掉一种复读后又复读替代写法 | 补丁痕迹不均匀，能删则删，能换通道就换通道 |
| 极短孤句成串 | 移动端一句一段正常；只拦 12 字内碎句连续 ≥4 或段长同构 |
| 同一配角连续吐槽 | 每章 ≤3，重要节点允许一句没人接的话 |
| “半/一点/几分”虚量词高频 | 同字每章 ≤4，具体物件如半袋米不计 |
| 物件立刻回应主角言行 | 每章 ≤4，至少一次延迟、一次缺席/静默，允许抢拍但不等距 |
| 主配角都说警句 | 连续警句式应答 ≤3 回合，双人语域必须可区分 |
| 同一连载装置连用 | 开头/结尾装置逐章登记，同一装置连续 ≤2 章 |
| 停家族/手滑三连/章尾显字 3/3 | 作为补丁复读、刻意失误复读、装置复读的实锤案例处理 |

## 段落级真重复（发布系统级别）

**这是量化工具容易漏的维度，但发布系统（番茄、起点、晋江等）会检测。**

**典型 AI 现象**：
- 长文生成时**自我抄袭**——同一段故事在 Ch8 末尾写了，Ch9 开头又复述了一遍（生成时为了"上下文连续"，把上一段的关键句重复写了一遍）
- 同一通对话里，关键台词在前后两处一字不差地复述
- 同一场景、同一动作链，间隔 200-500 字符后逐字重写

**检测方法**：
```bash
# 段落级独立重复（不是 n-gram 滑窗）
python3 quality/audit/scripts/paragraph_dup.py <文件>

# 全文量化信号（含段落级）
python3 quality/audit/scripts/text_signals.py <文件>
```

**编辑原则**：
- **删掉第二次出现**（保留第一次的"埋设"）
- **不能两边都"略作修改"**——发布系统会算"高相似度"，一样过不了
- 12 字级的"品牌母题"重复（同一个金句/Slogan/口头禅在不同场景的有意回扣）不算冗余

## 朱雀/AIGC 高分时的强信号复核

腾讯朱雀或其他外部检测给出高分时，不要只看“套路词”。优先复核这些最容易把短篇推高到 AIGC 高风险的信号：

| 强信号 | 表现 | 回改优先级 |
|---|---|---|
| 句长过平 | 连续 8-12 句都在相似长度，段落像等宽铺开 | 最高：拆短句、插动作打断、保留一两句不完整反应 |
| 解释归纳腔 | “这让她意识到 / 不再是而是 / 这意味着 / 终于明白”反复出现 | 最高：删结论，用动作、台词反应、误会后果替代 |
| 情绪命名 | “复杂情绪 / 难以言喻 / 五味杂陈”承担关键情绪 | 高：改成身体动作、物件处理、场景错位 |
| 平滑转场 | “然而 / 与此同时 / 随后 / 没过多久”机械串段 | 高：用具体动作或对话承接，减少万能连接词 |
| 工程词泄漏 | “本章 / 爽点 / 关系升级 / 伏笔回收 / 情绪爆发”进入正文 | 阻断：直接删改，正文不得出现制作术语 |
| 段落复述 | 同一段解释或对话在不同位置复现 | 阻断：删第二处，不做同义改写 |
| 过度完满 | 每个情绪点都被总结成道理，章末像作文收束 | 中高：留下未说透的动作、尴尬、误判或新问题 |
| 金句问号钩子 | 章末用"真正的选择是什么？""命运会给谁答案？"收束 | 中高：改成物件变化、新事实、未完成动作或具体追问 |
| 对白传送带 | 连续 6-8 个短段由不同人物依次提问、补背景、解围、宣布下一步 | 阻断：删不必发言者与不必当场说的信息，转入 POV 选择或现场余波 |
| POV 内在体验薄 | 主角只到场、观察、执行，刺激后补微动作但判断不变 | 阻断：补“刺激-误判/调节-选择变化-关系余波”的因果，不贴情绪标签 |

**朱雀 `0.8252` 的处理**：

- 记为 `腾讯朱雀AIGC值：0.8252（82.52%）`。
- 在当前严格 `<4%` 交付闸门下直接判“不通过”；`4%` 也不通过。
- 回改不要只替换近义词；优先改句法节奏、解释归纳腔、段落复述和角色声口。

**朱雀整章单片段报告的处理**：

- 若报告把约 3000 字章节作为 1 个片段，且显示“疑似AI占比 100%”或片段 AIGC 高于 50%，本地审核必须把它视为整章层面的高风险硬信号。
- 叙事人工锚点只能帮助复核误伤，不能覆盖整章单片段疑似 AI；报告中要同时列出外部片段值和本地 `zhuque_segment_proxy`。
- 这类问题通常不是一两个词、标点或比喻造成，而是整章功能过平：段落都在做同一种“观察-判断-记录-推进”，缺少对话摩擦、错误判断、物件后果和规则代价的自然换挡。
- 回改时先重排场景功能，再细修句子；禁止用脏码、随机清单、拟声长串或稀有名词堆叠扰动检测。

## 本地自研 AIGC 值

发布流程会自动运行：

```bash
python3 quality/audit/scripts/aigc_value.py <正文路径> --target 4
```

它输出 `自研AIGC值: 0.xxxx (x%)`，即最终 `AI占比`，引擎名为 `codex-local-aigc-v4`。该值按三层合成：近年检测器代理层占 85%，朱雀四维综合占 10%，既有 AI 味启发式占 5%。小说正文的 `narrative_scene` 锚点只软校准曲线和风格误伤，不提供固定最终低分；技术摘要/说明文的 `technical_expository` 锚点可用于校准，但不能覆盖当前严格 `<4%` 的项目门槛。若存在脏码、真重复或工程词泄漏，锚点不会启用。

四维权重：

- 突发性 burstiness：30%
- 困惑度代理 perplexity_proxy：25%
- 结构指纹 structure_fingerprint：25%
- 跨段一致性 cross_paragraph_consistency：20%

近年检测器代理层：

- 弱语言模型一致性：近似 DetectGPT/Fast-DetectGPT 的概率曲线思路，检查句级 unigram/bigram 惊讶度是否“过稳且过低”。
- 局部熵/TTR 波动：检查滑窗字熵、用字多样度是否跨段过稳，且整体多样度不足。
- 风格计量/可读性：检查句长分布是否集中、标点和对话表层是否单一。
- 语义平滑/概括腔：检查抽象概括、情绪命名是否压过动作、物件和感官锚点。
- 语意困惑度：检查句子是否长期承担同一功能，缺少自然的局部语义换挡。
- 叙事动力：检查对白传送带、动作开场标签同构、对白长度过齐、POV 内在体验薄、流程语汇过密和情绪范围过平。
- 朱雀式分片代理：对 1800-3600 可见字符按整章单段处理；整章疑似 AI 时，最终 AIGC 取分片风险下限，不能被局部人工锚点或全文平均稀释。

该值由以下信号合成：

- 句长 CV、段长 CV：越均匀越可疑。
- 套路措辞密度：时间锚点、微表情、情绪命名、金句升华、陈词意象。
- 解释归纳腔：`这让她意识到`、`不仅仅是/更是`、`终于明白` 等。
- 平滑转场：`然而`、`与此同时`、`随后`、`没过多久` 等机械串段。
- 工程词泄漏：`本章`、`细纲`、`爽点`、`关系升级` 等正文禁用词。
- 段落/句子复述：完全重复或高相似段落是阻断信号。
- 人味反向信号：对话比例、短句比例、具体物/数字密度、段落疏密变化。
- 高质人工样本锚点：小说正文看句长/段长疏密、对话声口、物件/动作/感官密度、标点层次、低套路密度；技术说明文看领域术语、英文/缩写词、摘要式长句承载。锚点必须让内容更成立，不能靠垃圾数据绕检。

回改原则：不要用“随机替换近义词”对付检测；优先改场景承载方式。让每段承担不同功能：动作推进、对话误解、物件细节、信息揭示、沉默反应交替出现；避免整章都用同一种平滑叙述腔。

内容优先原则：不要为了降低 AIGC 值插入无语义 OCR 脏码、随机汉字串、稀有神怪名词/怪字堆叠、长串拟声、词表式清单或其他绕检噪声。正常的残缺票据、聊天断句、设备误识别必须短、可读、能揭示规则或推进人物选择；否则按内容完整性失败处理。

服务端会把这个分数接入章节过审和终稿 quality gate。也就是说，审核报告手写“通过”不够；正文自动检测未过时，章节仍会被打回回改。

## 评分与置信度

- 六维各打 0–5 分
- 默认等权合成：`AI创作度% = (六维总分 / 30) × 100`，表述为 ±10% 区间
- 文本 < 300 字一律低置信度
- 套路化类型文（言情/玄幻）天然套路词多，必须靠维度 4（细节）和维度 6（声音）来判别
- **段落级真重复是 AI 起草的强信号**（人类写作极少出现）
- `aigc_value.py` 的自研AIGC值高于目标值时，服务端会自动判未过
- `text_signals.py` 的本地AI味风险分 > 35/100 时，发布流程应先回改再复审
