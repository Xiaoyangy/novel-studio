# Storycraft Emotion And Visual Engine

This note records the reusable basis for the prewrite storycraft fields generated by `--zero-init` and required by `plan_chapter`.

## Research Mapping

- Motivation: self-determination theory treats autonomy, competence and relatedness as basic psychological needs. The pipeline maps this into `emotional_logic`, `relationship_emotion_arcs`, and `CharacterDecisionFrame`, so choices are not only plot reactions.
- Emotion: appraisal theory treats emotions as responses to how events affect goals, resources, norms and coping options. The pipeline maps this into `goal_appraisal`, `boundary_threat`, `regulation_strategy`, `defense_mechanism`, and `emotion_led_action`.
- Romance/relationship arcs: relationship believability needs both fit and conflict. The pipeline maps this into `intimacy_stage`, `trust_debt`, `power_balance`, `boundary`, `romance_potential`, and `next_emotional_beat`.
- Visual design: character readability starts from silhouette, shape, costume, color, props and posture. The pipeline maps this into `visual_design`, including `silhouette`, `face_and_hair`, `clothing_style`, `color_palette`, `body_language`, `signature_object`, `status_wear`, and `change_rule`.

## Engineering Rules

- Every important character needs emotional logic, not just the protagonist.
- Every relationship that can change behavior needs a relationship emotion arc; no sudden loyalty, intimacy, betrayal, or romance without staged emotion.
- Every introduced character needs a visual identity that can be used in prose to carry scene information, not a static appearance paragraph.
- These fields are prewrite constraints. They can be refined by web search, current chapter state, or RAG, but they should not be omitted.
