# World Background Engine

This note records the reusable prewrite world-background layer used by `zero-init` and `plan_chapter`.

## Research Mapping

- SFWA worldbuilding questions emphasize history, geography, calendar, trade, conflicts, borders, crops, water and how people make a living. Map these to `world_background_layers`, `ritual_calendar`, `structural_resources` and `conflict_web`.
- SFWA secondary-world guidance flags politics and economics as things the writer must know even when not stated. Map this to `social_institution`, `economic_resource`, `hidden_rule_pressure` and `structural_resources`.
- Writing Excuses frames worldbuilding details as rules that create character stakes, and notes that different characters know different parts of the world. Map this to `information_asymmetry`, `narrative_tension_matrix` and POV boundaries.
- Sanderson's magic-system guidance stresses limitations, costs and boundaries. Map this to `cosmology_checks` so magic, weird contracts, AI or technology cannot become a cheat code.
- Information-gap craft requires consequence and a reveal path. Map this to `information_asymmetry.reveal_condition` and `tension_function`.
- Multi-antagonist/conflict craft warns against scattered conflict and asks each antagonist or pressure node to have a personal agenda tied back to the main conflict. Map this to `conflict_web`.

## Engineering Contract

`plan_chapter.causal_simulation` must include:

- `world_background_layers`
- `information_asymmetry`
- `hidden_rule_pressure`
- `social_mood_rumors`
- `ritual_calendar`
- `structural_resources`
- `cosmology_checks`
- `conflict_web`
- `narrative_tension_matrix`

These fields are not lore dumps. Each one must explain how the world changes what a character can know, want, risk, buy, hide, violate or pay.

`meta/world_background_plan.*` is generated by `--zero-init`, indexed by zero-init RAG, exposed through `reference_pack.references.world_background_plan`, and required by first-chapter readiness checks.
