从一句话指令
到 100 章完整长篇 的全链路

novel-studio 是一个 Go + LLM 长篇小说创作引擎。它把「前提 → 大纲 → 人物 → 世界规则 → 第一章 → 第 N 章 → 全文导出」拆成可断点续跑的流水线，让多个 AI Agent 分工协作，并通过本地 RAG（千问 Embedding + Qdrant）保持长程一致性。

5 coordinator / architect / writer / editor / reviewer
save_foundation type
9 含 expand_arc / append_volume

01 / ENTRY

三种入口

novel-studio 提供了三种主入口，覆盖「从零起手」「复用旧稿」「逐步澄清」三种场景。

Pipeline 流水线 主路径
最常用。从 prompt.md / inline prompt 启动，自动跑完整条 write→review→rewrite→export 链路。状态写到 meta/pipeline.json，可断点续跑。 
# 写法 A：inline prompt
./novel-studio --pipeline --prompt "写一部诡异末日..."

# 写法 B：从 prompt 文件
./novel-studio --pipeline --prompt-file prompt.md

# 写完第 10 章后停
./novel-studio --pipeline --prompt-file prompt.md \
  --stages write,review,rewrite --write-to 10
Cocreate 多轮澄清 可选
不直接写正文，先多轮对话把创作意图聊清楚，定稿 prompt 后再进入 pipeline。需要 TTY。 
# 多轮澄清需求
./novel-studio --cocreate

# 等价写法：显式列出 cocreate 阶段
./novel-studio --pipeline --stages cocreate,write,...
通常用 --pipeline 入口，--cocreate 是 TTY 下的快捷方式。
02 / PIPELINE

五阶段流水线

默认阶段 write → review → rewrite → deliver，可选前置 cocreate。每个阶段写盘到 meta/，可断点续跑；阶段内部还有更细的按章恢复（两层恢复叠加）。

0
Cocreate
cocreate
多轮澄清创作意图，定稿 --prompt。
1
Write
write
plan → draft → check → commit，循环跑完所有章节。
2
Review
review
Editor / Reviewer 评审每章，输出红旗 / 黄旗。
3
Rewrite
rewrite
按 review 反馈重写，最多 3 轮红旗闭环。
4
Export
export
合并已完成章节，输出 .txt / .epub。

03 / FOUNDATION

Foundation 五件套

所有产出的根基。由 Architect agent 通过 save_foundation 工具落盘，是写作阶段的硬前置。foundation 缺失时 --zero-init 和 pipeline 都会拒绝继续。

Premise
故事前提（一段话 / Markdown）
premise.md
Outline
单层大纲（章列表）或分层大纲（卷/弧/章骨架）
outline.md · layered_outline.md
Characters
人物系统（每项可带 big_five / 价值观 / 能力等心理画像）
characters.md
World Rules
世界规则（可标 visibility: formal/informal/secret，source: 朝廷/江湖/...）
world_rules.md
Book World
本书世界（地图、地点、势力图谱、protagonist_position）
book_world.md
save_foundation 工具 · 9 种 type
Architect 专用
type 写入文件 用途
premise premise.md / premise.json 故事前提（一段 Markdown）。自动从内容里抽书名
outline outline.md / outline.json 单层大纲（章列表）。写作阶段禁用，只能用 expand_arc 增量
layered_outline layered_outline.md / .json 分层大纲（卷 → 弧 → 章节骨架）
characters characters.md / .json 人物。每项可带 big_five 五维 / 依恋类型 / 认知偏差
world_rules world_rules.md / .json 规则。每条带 visibility（formal/informal/secret）和 source
book_world book_world.md / .json 本书世界。faction 可带 stance/internal_tension，relation 可带 conflict_type
expand_arc 细化某卷某弧 写作阶段增量展开骨架弧，需要 volume + arc 参数
append_volume 追加新卷并展开 写作阶段追加下一卷（content 传完整 VolumeOutline JSON）
complete_book Phase=Complete 宣告全书完结。必须通过终卷判定清单 + 无返工队列
subagent_guards.go:85 强制要求：architect 本轮必须落盘至少一次 save_foundation，否则不允许结束。
04 / WRITE LOOP

章节写作四步循环

write 阶段不是一次性产出，而是按章循环跑四步，每章产物落到 chapters/NN.md + 检查点写到 meta/checkpoints.jsonl。

01

Plan

plan_chapter

基于当前进度台账、人物续用参考、零章资产，生成下一章的 chapter_plan。

02

Draft

draft_chapter

调 Writer agent，按 plan 产出正文 draft。RAG 检索相关素材喂进上下文。

03

Check

check_consistency

一致性 lint（methodology_lints / consistency_lints / chapter_guard）。

04

Commit

commit_chapter

落盘 chapters/NN.md、追加检查点、增量沉淀 RAG、推进 progress。

commit_chapter 副作用链
关键路径
主线
写 chapters/NN.md → 追加 meta/checkpoints.jsonl → 推进 progress.json
rewrite
先把旧版备份成 chapters/NN.md.pre-rewrite.md
RAG
增量沉淀本章节入 embedding + Qdrant
下一章
生成 next_chapter_plan（人物续用 / 伏笔 / 资源账本）
⚠️ 工程设计意图与实现的已知差异：
chapter_metrics / chapter_world_deltas / character_stage 三类元数据只在 rewrite 路径写盘，normal commit 路径会漏接——这是工程里的设计意图与 code 行为不一致点（详见 commit_chapter.go）。
05 / AGENTS

多 Agent 分工

五个 Agent 角色按职责分离，每个角色独立配置 provider / model / reasoning_effort，并支持角色级 fallbacks 故障转移（主路挂掉自动切兜底，恢复后自动回主）。

coordinator
调度 / 路由 / 阶段切换
effort: medium
architect
设计 / save_foundation / expand_arc
effort: high 唯一调 save_foundation
writer
正文 / 重写正文
effort: high 章节主产能
editor
单章评审 / 反馈
effort: medium
reviewer
盲测 / pairwise / ai_voice
effort: max 异族裁判
角色级 Failover 范式
Task 065 异族裁判

LLM judge 对同族输出有 75–84% 自我偏好——writer 与 reviewer 建议用不同模型家族。请求级 failover：主 provider 失败（宕机 / 限流 / 余额不足）自动切兜底，恢复后自动回主。 

// ~/.novel-studio/config.json
{
  "roles": {
    "writer": {
      "provider": "anthropic",
      "model": "claude-opus-4-8",
      "reasoning_effort": "high",
      "fallbacks": [
        { "provider": "cc-codex-proxy", "model": "gpt-5.5" },
        { "provider": "minimax", "model": "MiniMax-M3[1M]" }
      ]
    },
    "reviewer": {
      "provider": "minimax",
      "model": "MiniMax-M3[1M]",
      "reasoning_effort": "max"
    }
  }
}
06 / RAG

本地 RAG 子系统

长篇小说动辄 100 章，单次上下文塞不下所有素材。RAG 把 foundation、零章资产、已写章节按需召回喂给 Writer，保证 100 章后风格 / 设定不漂移。

源材料

sources
  • foundation 五件套
  • 零章 11 类资产
  • 已写章节 + 元数据
  • reviews / drafts
  • 拆文库 / 对标库

Embedding

llama-server :18434

  • 1024 维 · 8K 上下文
  • 本地自动拉起 + 健康等待
  • OpenAI 兼容 /v1/embeddings

检索

keyword + embedding

./novel-studio --build-rag \
  --dir data/runs/鬼城/output/novel
增量沉淀
commit_chapter · review · rewrite 三个动作都会触发增量沉淀。新章节写完自动入向量库，下一章写作时召回相关素材。
⚠️ zero_init 阶段会重建 RAG 白名单索引（只索引本项目设定 + 零章资产，不索引旧正文/审稿/备份）。
07 / ZERO INIT

零章写前推演

--zero-init 是 foundation 之后的第二个固定动作。它基于已有的 premise / outline / characters / world_rules / book_world，推演出写作前需要的全部元数据资产。

11 类零章产物
下游：plan_chapter / draft_chapter
产物 作用
book_world.json/.md完整本书世界（若 foundation 缺）
meta/world_foundation.json/.md铁律 + 开局时间 + 过去时间线（角色未获得改变规则能力前不得突破）
meta/characters/<角色>/dossier.{json,md}每角色独立档案（主角未通信/未见证/无证据时不能知道）
meta/initial_character_dynamics.{json,md}初始人物动态（含 voice_logic）
meta/initial_resource_ledger.{json,md}初始资源账本（已入账 vs 待确认提案）
relationship_state.initial.{json,md}初始人物关系契约
foreshadow_ledger.initial.{json,md}初始伏笔账本（出现章 / 误导方向 / 回收方式 / 风险等级）
meta/prewrite_storycraft_plan.{json,md}写法工艺（arc tests / dialogue blueprints / voice cards / reader reward）
meta/world_background_plan.{json,md}世界背景层次 + 隐藏规则压力 + 仪式日历
meta/crowd_role_policy.{json,md}捧场/凑数角色策略（默认不入关键台账）
meta/character_return_plan.{json,md}人物回归与续用规划
drafts/01.zero_init.plan.json + meta/ch01_zero_init_plan.md第一章推演草案
meta/zero_chapter_context_manifest.{json,md}零章上下文清单（含 RAG 来源白名单/黑名单）
meta/initial_review_lessons.md初始评审教训沉淀
meta/simulation_restart_policy.{json,md}推演重启策略（旧章节仅作种子，新事实必须本轮推演）

./novel-studio --zero-init --check

./novel-studio --zero-init \

08 / SIDE SYSTEMS

辅助子系统

除了主链路，工程还配了一组「让你不用蹲在终端」的辅助能力。

Dashboard 浏览器看板
本地起一个 HTTP 服务，浏览器里看项目进度 / 章节 / 评审 / 元数据。
novel-studio service start
novel-studio service open
novel-studio service status
Notify 通知
完成后推送到桌面 / 手机。事件过滤：run_end / repeat / budget。
// Bark (iOS)
"command": "curl ... https://api.day.app/<key>"

// ntfy (iOS/Android)
"command": "curl ... ntfy.sh/<topic>"
Budget 预算熔断
累计花费达到 warn_ratio 告警，100% 优雅停机。挂机保险丝。
{
  "budget": {
    "book_usd": 50,
    "warn_ratio": 0.8,
    "hard_stop": false
  }
}
Steer 干预指令
不打断当前 run，排队一条指令，下次启动生效。
novel-studio --steer "第 5 章加一个反转"
Cocreate 多轮澄清
先聊清楚再开写。需要 TTY，CI / 远程场景走 --pipeline。
novel-studio --cocreate
Context 压缩
按窗口大小自动触发压缩；context_window 可把名义 1M 钉到 300k 提前压。
{
  "context_window": 300000
}
Check LLM 连通
正式跑前先确认 provider 能用。
novel-studio --check
novel-studio --check --role writer
Skills / 拆文库
内置 story skills + 工程内 拆文库/ 参考素材，可导入 RAG。
novel-studio skills list
novel-studio skills export --to ./skills
