# 百合（GL）故事圣经模板 · 约 1.5–2 万字符番茄短篇专用

> 配套文件：`skills/baiheprompt.md`（方法论）+ `skills/小说写作提示词模板.md`（通用四阶段流程）。
> **用法**：开书时按本模板填一份初版（`故事圣经 v1`），**每写一章前读取、写完后立即回写更新**。这是多章节强关联的技术核心——模型每次调用无记忆，全靠这份文件兜住一致性。
> 复制下方全部内容到 `story_bible_<书名>.md`，把 `〔 〕` 占位符替换为真实值，删掉用不到的行。

〔谁（攻+受）+ 因为什么被绑定 + 要跨过什么阻碍 + 走向什么结局〕

## 13. 番茄合规备忘（开写前贴在显眼处）

- [ ] 香艳/激情戏一律**虚写、转场留白**，不写性器官/露骨过程。
- [ ] 无血缘「骨科」必须**明确坐实无血缘**（基因报告/名义姐妹）。
- [ ] 校园线**不写未成年暴力/霸凌过程/自残自杀细节**，必要时改成年/侧面交代。
- [ ] 穿越/重生/算命只作设定，**不渲染改命玄学、不美化迷信**。
- [ ] 无任何站外导流、联系方式、拼音缩写、emoji、颜文字、装饰符号。
- [ ] 每章过：通用 16 条 + 合规 5 问 + 自校 5 步 + `baiheprompt.md` §八 百合 8 条。
- [ ] **完稿后定稿流水线**（见 `baiheprompt.md` §★）：⓪ **字数核验**（全书字符数 `[15000, 20000)`，`wc -m`；每章 `[1500, 2400]` 字符；不够按 §〇「六个增肥位」扩写）→ ⓪⓪ **整本降 AI 味精修**（跑 §★「B. 6 维对照 + C. 自检三问」，终版 AI 创作度 ≤ 5%，六维合计 ≤ 1，段落级重复为 0）→ ① `typo_scan.py` 第一轮改错别字 → ② 出审核报告 → ③ 按审核回改正文 → ④ **`typo_scan.py` 二次复核兜底**，两轮均通过才定稿。
