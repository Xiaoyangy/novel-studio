# deal_paper_summry —— 批量拆书 → 题材写作指导生成器

一条龙把「一文件夹原始小说」变成「一份可直接喂给写作模型的题材写作指导 Skill」。

**两个阶段，顺序不可乱：**
1. **阶段 A · 逐本拆解**：把目标文件夹下每本小说读成一份结构化摘要（8 大模块），落到 `data/generated-output/<题材>_summaries/<书名>.md`。
2. **阶段 B · 聚合生成**：所有摘要齐了之后，横向对比、提炼共性，结合 `skills/fanqie-novel-template/` 的通用骨架，产出 `skills/<题材>-writing-guide/SKILL.md`——一份专精该题材的写作指导 Skill。

> `skills/fanqie-novel-template/` 是阶段 B 的通用母版；`skills/baiheprompt.md` 和 `skills/nvpin_shuangnanzhuprompt.md` 是题材专精样例。阶段 B 要做的是把「拆 → 聚 → 成 Skill」的流程对**任意题材文件夹**复现。

---

## 输入约定（运行前先确认）

目标文件夹结构（`jjwxc_extract.py` 的产物即此结构）：

```
data/reference-library/<题材>/     ← 题材文件夹，<题材> 如 nvpin_shuangnanzhu / shuangwen
├── <书名A>/
│   ├── 001_第1章_xxx.txt          ← 分章纯文本，文件头通常含「《书名》/作者/章节名」
│   ├── 002_第2章_xxx.txt
│   └── ...
├── <书名B>/
└── ...
```

运行前先跑一句盘点，确认有几本、各本几章：

```bash
TARGET=data/reference-library/nvpin_shuangnanzhu  # ← 改成实际题材文件夹
for d in "$TARGET"/*/; do
  echo "$(ls "$d"*.txt 2>/dev/null | wc -l | tr -d ' ') 章  -  ${d#$TARGET/}"
done
```

- `<题材>` 取文件夹名（如 `nvpin_shuangnanzhu`），阶段 B 的产物 `skills/<题材>-writing-guide/SKILL.md` 与摘要目录 `data/generated-output/<题材>_summaries/` 都用它命名。若需要兼容旧流程，可额外生成 `skills/<题材>prompt.md`。
- 若用户没指定文件夹，问清楚拆哪个 `data/reference-library/*` 目录再开工。

---

## 阶段 A · 逐本拆解（每本一份摘要）

**对题材文件夹里的每一本书**，依次：

### A1. 读全书（长则抽样）
- 每本的所有 `*.txt` 按章节号排序读入。文件头部的「《书名》/作者/章节名/来源」要识别出来填基本信息。
- **若全书很长（章节多、总字数大）**：不必逐字精读全文。**通读开头 3 章 + 结尾 2 章**，中段按每 3–5 章抽 1 章精读，其余扫读抓主线/钩子/伏笔。在摘要里标注「（基于抽样）」。
- 字数核验可借现成命令：`wc -m "$TARGET/<书名>"/*.txt` 看总字符量级。

### A2. 写摘要文件
落到 `data/generated-output/<题材>_summaries/<书名>.md`（先 `mkdir -p data/generated-output/<题材>_summaries`）。**严格按下面 8 个模块成文**，每个模块都要「具体到能照着仿写」，忌空泛套话：

```markdown
# 《<书名>》拆解摘要

## 1. 基本信息
- **书名 / 作者**：
- **题材与细分类型**：（如 女频·双男主·破镜重圆 / 娱乐圈 / 校园 …）
- **体量**：章节数、约 X 万字符（`wc -m`）、是否完结/试读
- **一句话核心设定**：（≤30 字，点破全书最核心的「钩子+反差」）
- **CP / 人物配置**：攻受位 or 双强 / 双向 / 单箭头 …
- **结局类型**：HE / BE / 开放
- **平台与标签**：来源平台 + 3–5 个题材标签

## 2. 设计思路
- **核心创意 / 差异化卖点**：这本书凭什么不撞款
- **类型定位**：踩的是哪个细分赛道 + 读者预期
- **情感引擎**：双向暗恋 / 救赎 / 原生家庭创伤 / 追妻火葬场 … 哪一组在驱动
- **矛盾引擎**：核心冲突来源（身份差 / 误会 / 外部反派 / 立场对立 …），如何层层叠加
- **节奏设计**：开篇炸点位置、爽虐配比、黄金三章如何留人、章节长度规律

## 3. 主角配角模版
- **主角 A（攻 / 强）模版**：人设标签 + 反差线（外冷内热 / 钝感 / 偏执 …）+ 高频言行
- **主角 B（受 / 弱）模版**：人设标签 + 反差线 + 「小动物化 / 钓系」等萌点手法
- **双主互动公式**：称谓演进、肢体距离升级轨道、声口差异
- **关键配角**：每个配角承担的功能（工具人 / 反派 / 助攻 / 镜像对照），可复用的配角位

## 4. 故事钩子
- **开篇黄金前三行**：原文摘录 + 拆它为什么抓人
- **章末钩子库**：摘 3–5 个章末钩子原句，标注类型（悬念 / 反转 / 暧昧升级 / 危机）
- **黄金三章结构**：第 1/2/3 章各抛了什么、解决了什么、留了什么

## 5. 伏笔与回收
- **埋点—回收表**：| 伏笔 | 埋设章节 | 回收章节 | 作用 |（至少 3 组）
- **意象 / 象征体系**：反复出现的物件/动作/台词，及其情感承载与回收点

## 6. 吸引人的点
- **爽点库**：本书用了哪些爽点（打脸 / 逆袭 / 独宠 / 实力碾压 …），各摘 1 例
- **虐点库**：误会 / 分离 / 牺牲 / 错过 …，各摘 1 例
- **磕点库（CP 糖点）**：肢体/称谓/告白的渐进暧昧，摘最出圈的 2–3 个名场面
- **一句话总结「读者为什么追完」**

## 7. 标题命名设计
- **书名拆解**：用了什么命名套路（悬念式 / 人设式 / 反差式 / 第一人称喊话式 …），仿写 2 个同类书名
- **章节标题规律**：是否分章名、长短、口语化程度，摘 3 个代表
- **简介 / 文案套路**：开头第一句怎么钩、用了哪个文案公式、话题标签怎么挂

## 8. 可复用要点（供阶段 B 聚合）
- 用 3–5 条 bullet 提炼「这本最值得抄进方法论的招」，每条一句话、可直接迁移。
```

### A3. 进度记录
- 每拆完一本，在 `data/generated-output/<题材>_summaries/_index.md` 追加一行：`- [《书名》](书名.md) — 一句话亮点`。
- 已存在的摘要文件**跳过**（可断点续拆）。**全部本子都有摘要后，才进入阶段 B。**

---

## 阶段 B · 聚合生成「<题材>写作指导 Skill」

所有摘要齐了之后，把它们横向读一遍，提炼**跨书共性**，按 `skills/fanqie-novel-template/references/genre-skill-blueprint.md` 的结构产出 `skills/<题材>-writing-guide/SKILL.md`。

### B1. 聚合分析（动笔前先做）
读 `data/generated-output/<题材>_summaries/*.md` 全部 8 个模块，按维度归并、统计高频项：
- **细分类型分布** → 该题材主流赛道有哪几条
- **人设模版** → 把各书主角/配角模版聚成「攻方 N 种高频模版 / 受方 N 种高频模版」
- **情感引擎 & 矛盾引擎** → 哪几组组合出现最多 = 该题材命脉
- **开篇打法 & 章末钩子** → 归并成「钩子库」
- **伏笔/意象手法** → 归并成「意象体系方法论」
- **爽/虐/磕点** → 三个分类的「点库」
- **书名/章节/简介** → 命名与文案方法论 + 公式
- **共性雷区** → 从多本对比里反推「不能踩的坑」

### B2. 套用通用模板成文
先读 `skills/fanqie-novel-template/SKILL.md`、`skills/fanqie-novel-template/references/fanqie-short-workflow.md`、`skills/fanqie-novel-template/references/genre-skill-blueprint.md`，再把聚合出的题材方法论写进 `skills/<题材>-writing-guide/SKILL.md`。`skills/baiheprompt.md` 与 `skills/nvpin_shuangnanzhuprompt.md` 只作为题材专精写法参考，不再作为唯一母版。

建议保留这套结构（按题材增删）：

1. 顶部说明：本 Skill 继承 `skills/fanqie-novel-template/SKILL.md`，只补本题材专精方法论；列出聚合自哪几本书（`data/generated-output/<题材>_summaries/`）。
2. 必读材料：通用模板、摘要目录、review 技能、可选样例 prompt。
3. 赛道选择：主流细分类型清单，每本短篇最多 1 主 1 副。
4. 人设与关系动力：主角/配角高频模版、反差线、互动或对抗公式。
5. 情感 / 爽点 / 虐点 / 磕点或燃点引擎。
6. 开篇炸点与章末钩子库。
7. 伏笔、意象、信物或证据链体系。
8. 标题、金句、简介、标签方法论。
9. 平台合规注意与题材雷区。
10. AI 味 5% 交付闸门：列出本题材最容易触发 AI 味的信号和改写动作。
11. 写作自检清单。
12. 一页速查 SOP。

**成文要求**：
- 每条方法论尽量**带样本出处**（「见《X 书》开篇」），让指导可溯源、不空谈。
- 凡能给「可直接套写的模板/公式/原句范例」的，就给——这是指导 Skill 的价值所在。
- 字数计量、三件套、降 AI 味、错别字自校等通用机制**引用** `skills/fanqie-novel-template`、`skills/review`、`quality/audit/scripts/`，不重复造。
- 新生成的题材 Skill 必须继承“终版 AI 味 5% 以下”硬闸门；不得保留 35%、40% 等旧宽松阈值。
- 文件头 YAML frontmatter 写清 `name`（如 `<题材>-writing-guide`）与 `description`（含触发词），并运行 skill 校验脚本。

### B3. 交付
- 产物：`skills/<题材>-writing-guide/SKILL.md`（指导 Skill）+ `data/generated-output/<题材>_summaries/`（全部拆解摘要，作为方法论的证据底稿）。如用户明确需要旧 prompt 文件，再额外输出 `skills/<题材>prompt.md`。
- 在回复里给一张小结表：拆了几本、聚出哪几条该题材命脉、生成了哪个 Skill 文件。

---

## 操作铁律

1. **先 A 后 B**：没把文件夹里每本都拆出摘要，不准进阶段 B 聚合。
2. **摘要要具体**：8 大模块里凡是要「原句 / 模板 / 表格」的地方，必须摘真实样本，不许写「比较精彩」这类空话。
3. **断点续拆**：摘要文件已存在就跳过；中途可停可续。
4. **可溯源**：阶段 B 每条方法论尽量挂样本出处，方便复盘与迭代。
5. **不污染**：摘要进 `data/generated-output/<题材>_summaries/`，指导 Skill 进 `skills/`，原始小说只读不改。
