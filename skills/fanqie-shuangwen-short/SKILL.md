# 番茄反转爽文短篇写作工具

使用本技能时，按“通用骨架 + 反转爽文专精”的顺序工作。这个方向先作为轻量方法 skill；如需更细赛道，可用 `skills/deal-paper-summry/SKILL.md` 对 `data/reference-library/shuangwen/` 或新样本目录做聚合升级。

## 必读材料

1. 先读 `skills/fanqie-novel-template/SKILL.md`；需要细节时读 `skills/fanqie-novel-template/references/fanqie-short-workflow.md`。
2. 参考样本与成稿：
   - `data/reference-library/shuangwen/*.md`：番茄爽文样本。
   - `data/generated-output/nanpin_shuangwen/*/故事圣经.md`：男频反转爽文成稿结构。
   - `data/generated-output/nvpin_shuangwen/*/故事圣经.md`：女频婚恋/职场逆袭爽文成稿结构。
3. 没有专属故事圣经时，使用 `skills/fanqie-novel-template/references/story-bible-template.md`，并把关系字段改成“主角 / 对手 / 助攻 / 证据链 / 打脸节点”。
4. 完稿质检时使用 `skills/review/SKILL.md` 和 `quality/audit/scripts/`。

## 赛道选择

短篇最多选 1 条主赛道 + 1 条副钩子：

- 婚恋背叛：出轨、替身、养鱼、家庭牺牲被辜负，核心是“我不再解释，直接抽身反制”。
- 职场维权：抄袭、抢功、PUA、性别偏见、资源倾斜，核心是证据链和公开碾压。
- 男频隐忍反杀：装窝囊、被轻视、旧物/能力/人脉反转，核心是压抑到爆发。
- 重生先知：前世惨局开篇亮出，2 章内完成避坑，后文用先知优势打脸。
- 都市情感反转：亲情、友情、爱情背叛合一，但反派数量要少，避免短篇铺不开。

## 执行要点

- 默认目录按受众选：`data/generated-output/nanpin_shuangwen/<书名>/` 或 `data/generated-output/nvpin_shuangwen/<书名>/`。
- 默认体量：用户给总字数时按 2100-3000 字/章反推章节；未给时沿用番茄短篇目标，全书 `wc -m` 总字符数 `[15000, 20000)`，通常约 5-10 章。
- 正文文件头按番茄格式写：书名、作品标签、主角人设、金句、简介、话题标签，再进入第一章；作品标签从通用词表中选择，不把爽文当作唯一标签。
- 开篇三行内必须出现背叛现场、重大羞辱、翻车证据或反常冷处理。
- 每章必须有一个爽点推进：取证、断供、离场、公开揭穿、对手自爆、第三方误判翻盘、资源归位。
- 证据链必须落地到具体物件：聊天记录、合同、录音、监控、论文原稿、旧表、转账、会议纪要。
- 结尾必须把“主角获得什么新秩序”写实，不只写反派倒霉。
- 终版必须通过 `fanqie-novel-template` 的 AI 味 5% 交付闸门；爽文最容易变成口号和整齐打脸清单，必须用具体证据、金额、时间戳、文件名和不完美口语压低模板感。
- 短篇正文禁止整齐模板扩写感：不能每章都按同一段式完成“被羞辱 + 找证据 + 反派跳脚 + 主角冷笑打脸”。每章至少落 2 个不可替换的证据细节、金额、时间戳、文件名、错字聊天记录或第三方误判；爽点要靠证据链和现场反应，不靠排比口号。

## 交付

交付以单章文件为准：`{书名}/第NN章_章名.md`、`故事圣经_<书名>.md`、`审核报告_<书名>.md`、`图片生成方案_<书名>.md`。`正文_<书名>.md` 只能作为所有单章审核通过后的合并稿；所有审稿订正必须先回写对应单章文件，审核报告必须写明 AI 味 5% 闸门已过。配图方案必须基于终版正文生成，至少包含封面图和 3 张关键场景图提示词。
