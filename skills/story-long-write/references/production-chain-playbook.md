# 长篇写作生产链蒸馏手册

此文件把 AI-Novel-Writing-Assistant 中可迁移到 story-long-write 的经验压缩成写作链路规则。它不是文风模板，而是判断资料、细纲、正文、审稿和返修分别该做什么的边界。

## 1. 分层边界

- 大纲 / 卷纲 / 细纲决定写什么：目标、冲突、场景顺序、章节钩子、下一章承接。
- 设定 / 角色 / 世界观 / 追踪文件决定事实是什么：角色当前状态、伏笔、时间线、势力边界、可用资源。
- 文风 / 写法资产决定怎么写：句长、停顿、对白潜台词、叙事距离、情绪外显、禁用表达。
- 对标 / 拆文 / RAG 资料决定参考什么：只学结构、节奏、情绪触发和写法，不搬人物、专名、桥段和原句。
- 质量检查决定问题在哪一层：表达问题修正文，细纲问题补细纲，事实问题修追踪或设定，连续结构失效才重排大纲。

优先级：已写正文事实和追踪文件 > 细纲 / 章节任务 > 用户硬要求 > 设定 / 角色 / 世界观 > 文风 / 写法规则 > 对标拆文参考 > 临时聊天建议。

## 2. 单章任务单

细纲要能直接变成正文任务单，至少回答：

- 本章目标：事件、关系、情绪或信息推进是什么。
- 承接上章：上一章留下的动作、状态、情绪余波。
- 铺向下章：结尾把读者带向哪个问题、危机或选择。
- 场景卡：地点、参与者、冲突、信息变化、可见后果。
- 参与者：只读取本章涉及角色和势力，不把全书角色全塞进写作上下文。
- 兑现 / 伏笔：本章兑现什么，推进什么，新埋什么。
- 禁忌：不能提前揭露的谜底、不能越界的规则、不能写成既成事实的待确认资源。

如果细纲只有标题和一句剧情概括，先补细纲，不要直接开写。

## 3. 写法资产

写法资产应按三层使用：

- 意图：用户要的读感，例如粗粝、克制、轻快、压迫、口语化。
- 合同：可执行规则，例如句长、断段、对白、情绪外显、反 AI 约束、样本锚点。
- 补丁：审稿发现的重复表达偏移，沉淀为下一章也要遵守的规则。

写法只管表达，不管角色事实和剧情走向。样本只能模仿节奏、取景和句法，不能复刻桥段、专名和句子。

## 4. 拆书与 RAG 资料

读取对标或知识库时，用本章任务驱动召回：角色、地点、势力、冲突、情绪目标、待兑现承诺、需要查证的事实。不要只按章节号或标题召回。

资料优先级：

1. 本书正文、摘要、追踪文件和设定。
2. 大纲、卷纲、细纲和章节任务。
3. 用户资料、知识库、参考资料。
4. 拆书结论：情绪模块、节奏、桥段技法、写法总结。
5. 对标原文锚点。

拆书结论进入正文前必须替换为本书的人物、场景、规则和代价；命中弱或来源不清时宁可不用。

## 5. 质量债务与返修分流

局部问题不等于整本书失败。先判断正文是否可用：

- 可用且只有轻微表达问题：记录并局部润色，继续下一章。
- 少量契约漏项可补：补当前章，不重写全书。
- 当前章没完成核心目标：重写当前章。
- 角色、世界、资源或时间线冲突：先修追踪 / 设定，再改受影响章节。
- 细纲或后续结构已失效：回到大纲 / 卷纲 / 细纲重排。

不要把事实错当成文笔问题；也不要把 AI 腔、句式疲劳这种表达问题扩大成全局重规划。

## 6. 反 AI 和语意困惑度

去 AI 味不是加脏码、稀有词或随机断句，而是让每段有真实承载：

- 连续抽象判断后，必须落到动作、物件、感官、对话或选择后果。
- 段落功能要换挡，不能总是概述、心理、转场同一套。
- 情绪不要只贴标签，要通过身体反应、误判、回避、临时改口、行动代价呈现。
- 对话不要每句都信息完整，允许沉默、偏题、抢话、误解和未说完，但必须服务人物和场景。

每次修复都尽量回写到细纲、追踪、文风或禁用规则中，避免下一章重复犯同一类问题。
