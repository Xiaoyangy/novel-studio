# 番茄短篇通用写作流程

本文件只写题材无关规则。题材专精 skill 可以覆盖章节配比、目录名、标签词和合规表达，但不应放松这些交付约束。终版 AI 味必须在 5% 以下，这是交付闸门，不是建议项。

## 1. 三件套 + 配图目录

每本书一个独立目录：

```text
<书名>/
├── 第01章_章名.md
├── 第02章_章名.md
├── ...
├── 故事圣经_<书名>.md
├── premise.md
├── characters.json
├── characters.md
├── world_rules.json
├── world_rules.md
├── book_world.json
├── book_world.md
├── outline.json
├── outline.md
├── layered_outline.json
├── layered_outline.md
├── timeline.json
├── timeline.md
├── relationship_state.json
├── relationship_state.md
├── foreshadow_ledger.json
├── foreshadow_ledger.md
├── compass.json
├── compass.md
├── 审核报告_<书名>.md
├── 图片生成方案_<书名>.md
├── images/             # 可选，保存已生成图片
└── 正文_<书名>.md   # 可选合并稿，所有单章审核通过后生成
```

若历史产物使用 `正文.md` / `故事圣经.md`，可以兼容；新工具优先使用统一设计包 + `{书名}/第NN章_章名.md` 单章文件作为主产物。短篇不省设计文件，只把 `layered_outline` 压缩为 1 卷 1 弧，把时间线、伏笔和关系变化压到篇内闭合。

## 2. 正文文件头

```markdown
# <书名>

作品标签：
- 类型：<标签1> / <标签2>
- 角色：<标签1> / <标签2> / <标签3>
- 情节：<标签1> / <标签2> / <标签3>
- 情绪：<标签1> / <标签2>
主角人设：<人物 A 核心身份 / 处境> VS <人物 B 核心身份 / 性格 / 功能>

> **<金句>**
>
> <作品简介>
>
> #标签1 #标签2 #标签3

第一章 <章节标题>

<正文>
```

- 金句 25 个汉字以内，不复述书名，要点破核心反差或情感落点。
- 简介 100-150 字，第一句就是钩子，末尾挂 2-4 个话题标签。
- 作品标签必须从 `work-tag-catalog.md` 选择：类型 1-3 个、角色 2-5 个、情节 2-5 个、情绪 1-3 个；双男主或双女主作品必须在「角色」里写入「双男主」或「双女主」。
- 主角人设必须使用 `A VS B` 对照格式，例如 `不被爱的富家千金 VS 温柔暖心哥哥`。
- 只有书名使用 `#`，章节标题用纯文本，不用 `##`。

## 2.1 反模板扩写闸门

短篇公式只用于确定情绪顺序和功能位，不能把正文写成“模板逐项扩写”。每章审核时必须检查：

- 不按固定段式复刻：禁止每章都呈现“开场冲突 → 回忆解释 → 台词爆发 → 章末金句”的同构节奏。
- 不写均匀段落：段长、对话密度、动作密度要随场景变化；高潮可以短促，沉淀可以稍长。
- 不用功能标签替代场景：正文里不能出现“爽点推进、关系升级、伏笔回收、情绪爆发”等工程词，也不能把这些词换皮成旁白解释。
- 每章至少有 2 个只属于本书/本章的具体物件、动作打断、口癖、错误信息、时间戳、金额、平台词或私人物件，不能只用通用情绪句。
- 章尾不要为了工整收口：不凑对仗、排比、押韵总结或“至此我终于明白”式归纳；用具体悬念、动作停顿或未说完的话收束。
- 如果读起来像把细纲每条平均扩成一段，本章不通过；回到当前章重写疏密、细节和声口，不进入下一章。

## 3. 体量门控

默认写番茄精短篇：

| 指标 | 硬下限 | 硬上限 | 推荐 |
|---|---:|---:|---:|
| 全书字符数 | 15000 | 20000 | 16000-18000 |
| 章节数 | 8 | 10 | 10 |
| 常规单章字符数 | 1500 | 2400 | 1700-2100 |

判定一律用 `wc -m` 总字符数，含标点、空行和 Markdown 标记。纯汉字数只作内部参考。

```bash
wc -m "data/generated-output/<题材>_novels/<书名>"/第*.md
python3 -c "import re,sys,pathlib;t=''.join(p.read_text(encoding='utf-8') for p in pathlib.Path(sys.argv[1]).glob('第*.md'));ch=len(t);hz=len(re.findall(r'[\u4e00-\u9fff]',t));print(f'总字符 {ch} | 纯汉字 {hz} | 占比 {hz/ch*100:.1f}%')" "data/generated-output/<题材>_novels/<书名>"
```

## 4. 逐章纪律

1. 写前读取统一设计包：`故事圣经_<书名>.md`、`premise.md`、`outline.json/md`、`timeline.json/md`、`characters.json/md`、`relationship_state.json/md`、`foreshadow_ledger.json/md`。
2. 一次只写一章。
3. 每章按“主场景 + 次场景 + 章末钩子”搭建。
4. 正文直接写入 `{书名}/第NN章_章名.md`，不要只追加到合并正文。
5. 写完立刻数本章字符数；不足就扩，超限就砍。
6. 立刻审核本章：题材硬规则、承上启下、AI 味信号、错别字、一致性。审核未通过时只修当前章，复审通过后才进入下一章。
7. 回写故事圣经索引和相关设计状态：本章摘要、字符数、伏笔状态、关系/爽点推进、下一章钩子、单章审核结论。

## 4.1 多小说并发与上下文保护

- 用户一次给出多个题材/设定时，按每本书拆分为独立任务。Codex 不会自动开 subagent；必须明确写出 `spawn N agents` 或“并行拆给 N 个 agent”，把不同小说显式分派给不同写作 agent，最多同时运行 5 个写作 agent。
- 多于 5 本时按批次排队：先启动 5 本，任一本完成、阻塞或暂停后，再把下一本 pending 小说补进空出的 agent 槽位。
- 每个 agent 只拿该书输入、该书目录、统一设计包、状态文件和当前章上下文，不读取其他小说正文。
- 单本内部不并发写章；必须逐章写、逐章审、逐章回写。
- 主会话只记录调度表、agent 槽位和每本最后审核结论，正文内容通过文件系统持久化，避免把多本长文本堆入同一上下文。

## 5. 通用增肥位

- 加一个带动作链的对话回合，而不是解释人物心理。
- 把核心情绪写成具体动作、停顿、物件触碰。
- 强化一个意象或信物的当章功能。
- 加一处反差细节，让人物设定落到场景里。
- 加一个配角打断、误判或递线索，但不抢主线。

## 6. 通用减肥位

- 删同一情绪的换皮复述。
- 删无功能寒暄和过渡脚手架。
- 删给读者讲课的第二人称解释。
- 删重复环境描写，只保留最有辨识度的物件。
- 删不影响后续回收的支线角色和支线伏笔。

## 7. 完稿流水线

顺序不可乱：

1. 字数核验：全书 `wc -m` 必须落在 `[15000, 20000)`。
2. 整本降 AI 味精修：逐章处理过度整洁、抽象抒情、解释过满、重复句式，目标是进入审核前已经接近人工稿。
3. 第一轮错别字：运行 `python3 quality/audit/scripts/typo_scan.py <正文路径>`，人工确认后回写。
4. AI 创作度审核：逐章运行 `quality/audit/scripts/aigc_value.py`、`quality/audit/scripts/text_signals.py`、`quality/audit/scripts/paragraph_dup.py` 与 `quality/audit/scripts/content_lint.py`，为每章生成 `reviews/NN_ai_gate.json` 与统一审核报告 `reviews/NN.md`，再按 `skills/review/SKILL.md` 写报告，并在结论里写明是否通过“AI 味 5% 交付闸门”。如果用户提供腾讯朱雀/AIGC 值，运行 `text_signals.py` 时追加 `--external-aigc <值>`；例如 `0.8252` 必须换算为 `82.52%`。
5. 按审核报告和 `reviews/` 机械门禁回改正文；只要自研AIGC值高于 5%、AI 创作度高于 5%、外部 AIGC 检测值高于 5%、本地AI味风险分高于 35/100、六维 AI 倾向分合计大于 1、任一单项大于 1、段落级重复未清零，或 `content_lint.py` 命中未处理的硬检问题，就必须继续重写并重跑审核。
6. 第二轮错别字复核，并在审核报告末尾记录复核结果。
7. 根据终版正文生成配图包：至少 1 张封面图 + 3 张关键场景图的图片提示词；若当前执行环境支持图片生成，实际生成图片并保存到 `images/`，在 `图片生成方案_<书名>.md` 记录路径。

## 8. AI 味 5% 交付闸门

终版交付前必须同时满足：

- 自研AIGC值 / AI占比不高于 5%；服务端会自动运行 `aigc_value.py` 检测正文，不能只靠审核报告手写通过。该值由 `codex-local-aigc-v3` 合成：近年检测器代理层、朱雀四维代理分、内容完整性/绕检噪声代理层和既有 AI 味启发式。
- AI 创作度结论不高于 5%。
- 腾讯朱雀/AIGC 外部检测值若提供，换算后不高于 5%；`0.8252` 记为 `82.52%`，直接未过。
- `text_signals.py` 的本地AI味风险分不高于 35/100。
- 六维 AI 倾向分合计不超过 1，且任一单项不得超过 1。
- `paragraph_dup.py` 的完全重复段落、高度相似段落、重复长句均为 0。
- `content_lint.py` 的内容逻辑、库存明喻、语义清晰、标点情绪层级问题已处理；error 不得交付，warning 必须逐条确认。
- 每章都存在 `reviews/NN_ai_gate.json` 与 `reviews/NN.md`，且机械门禁结论为通过。
- 套路措辞密度、句长 CV、重复片段等脚本信号不得出现“强烈 AI 倾向”的解释。
- 审核报告末尾追加记录：`AI 味 5% 闸门：已过 / 未过；未过则列出外部 AIGC 值、本地AI味风险分、回改章节和重跑结果`。

未通过时，不要给用户终版正文；回正文改稿，再重新走第 4-6 步。

## 9. 终版检查

- 单章正文文件完整存在，且每章已审核通过。
- 合并正文如需交付，只能由单章文件生成。
- 章节标题没有 `##`。
- 作品标签、主角人设、金句、简介、话题标签存在且符合格式和字数。
- 全书和单章字符数达标。
- 故事圣经伏笔全部回收或明确舍弃。
- 审核报告记录了脚本结果、回改动作、AI 味 5% 闸门和二次错别字复核。
- 图片生成方案存在，包含封面图、关键场景图、角色一致性说明、正向/负向提示词和图片保存路径。
