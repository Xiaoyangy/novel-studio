# 番茄小说通用模板

本技能只提供题材无关的工程骨架：目录结构、番茄短篇体量、统一设计包、逐章写作纪律、完稿审核和新题材 skill 生成规范。具体人设、关系动力、爽虐甜点、桥段库和合规表达由题材专精 skill 填充。

## 使用路径

1. 创建新题材写作工具：先用 `skills/deal-paper-summry/SKILL.md` 拆样本，再读 `references/genre-skill-blueprint.md`，生成 `skills/<题材>-writing-guide/SKILL.md`。
2. 写一本番茄短篇：先读 `references/fanqie-short-workflow.md`，再叠加对应题材 skill，例如 `skills/fanqie-baihe-short/SKILL.md`。
3. 建统一设计包：按 `docs/design-stage-workflow.md` 生成同名结构化文件，`故事圣经_<书名>.md` 只做可读总览和索引；没有题材专属模板时，使用 `references/story-bible-template.md` 补全可读总览。
4. 完稿质检：按通用定稿流水线执行，再调用 `skills/review/SKILL.md` 和 `quality/audit/scripts/`。

## 分层规则

- 通用模板负责“必须怎么交付”：三件套、配图包、文件头、字数、章节、逐章回写、审核流水线。
- 题材 skill 负责“这类故事写什么”：赛道、人设、关系、情感引擎、钩子、伏笔、点库、标题简介公式、雷区。
- `deal-paper-summry` 负责“从样本提炼题材方法论”：先逐本拆解，再按通用模板生成题材 skill。
- 不要在题材 skill 里整段复制通用规则；引用本技能即可，只覆盖题材差异。

## 通用硬约束

- 默认产物是番茄精短篇。若用户给预期总字数，按 `novel-studio` 统一单章 2100-3000 字反推大致章数；若未给总字数，平台短篇默认目标仍按全书 `wc -m` 总字符数 `[15000, 20000)` 估算，通常约 5-10 章，章数服务内容闭环，不为卡整数牺牲剧情。
- 每本书必须独立落盘到 `{书名}/` 目录；正文以单章文件为主产物，路径固定为 `{书名}/第NN章_章名.md`。不要把所有章节只写进一个合并正文。
- 设计包必须先保留：`premise`、`characters`、`world_rules`、`book_world`、`outline`、`layered_outline`、`timeline`、`relationship_state`、`foreshadow_ledger`、`compass` 和 `故事圣经_<书名>.md`。三件套仍要保留：`故事圣经_<书名>.md`、`审核报告_<书名>.md`、可选的 `正文_<书名>.md` 合并稿。合并稿只能在所有单章审核通过后生成，不能替代单章文件；完稿后还必须生成 `图片生成方案_<书名>.md`，记录封面图、关键场景图提示词和图片路径。
- 正文文件头必须包含书名、作品标签、主角人设、25 字以内金句、100-150 字简介和 2-4 个话题标签；作品标签只能从 `references/work-tag-catalog.md` 选择。
- 一次只写一章；每章写完立刻数本章字符数、运行本章审核、回写故事圣经索引和相关设计状态。单章按 2100-3000 字预算控制承载量，明显越界才回改；不要为了几十到几百字牺牲必要铺垫、人物选择或读者读感。只有本章审核达标后，才能进入下一章。
- 完稿后固定顺序：字数核验 -> 整本降 AI 味精修 -> 第一轮错别字 -> AI 创作度审核 -> 按报告回改 -> 第二轮错别字复核 -> 根据终版正文生成配图方案 / 图片。
- 终版 AI 味必须在 5% 以下；审核报告高于 5% 或段落级重复未清零时，不准交付。

## 多小说并发规则

当用户一次给出多个不同题材、书名或核心设定时，必须按“每个题材/书名 = 一本小说”拆分任务：

- 为每本小说创建独立目录、状态和统一设计包，不共享正文上下文。
- Codex 不会自动开 subagent；多书并行时必须在调度指令里明确写出 `spawn N agents` 或“并行拆给 N 个 agent”，并为每本小说显式分派一个独立 agent。**最多同时运行 5 个写作 agent**，超过 5 本就分批排队。每个 agent 只接收该小说的输入、目录、题材 skill、状态文件和当前阶段，不要把其他小说的设定塞进同一个上下文。
- 单本小说内部仍然串行：第 N 章必须写完、审核通过、故事圣经回写后，才写第 N+1 章。
- 主会话只保留调度表、agent 槽位、每本当前章号和审核结论；长正文、旧章节、详细设定都保存在文件系统，按需读取，避免上下文被压缩。

## AI 味 5% 交付闸门

终版审核必须满足三条：

1. `审核报告_<书名>.md` 的结论写明 AI 创作度不高于 5%。
2. 六维 AI 倾向分合计不超过 1，且任一单项不得超过 1。
3. `paragraph_dup.py` 检出的完全重复段落、高度相似段落和重复长句都为 0。

任一不满足时，回到正文逐章重写问题段，再重跑 `text_signals.py`、`paragraph_dup.py` 和错别字复核，直到通过。

## 新题材 Skill 产出标准

新题材 skill 至少包含：

1. 清晰 frontmatter：`name` 使用 hyphen-case，`description` 写明触发场景、题材、平台和产出目标。
2. 素材来源：列出 `data/generated-output/<题材>_summaries/` 的样本书和聚合依据。
3. 通用模板继承声明：引用本技能，不重复长篇通用规则。
4. 题材专精章节：赛道选择、人设模板、关系/爽点/虐点/磕点或燃点、钩子库、伏笔意象、标题简介公式、合规雷区。
5. 一页 SOP：从开书、建圣经、排章、扩写、审核、AI 味 5% 闸门到终版交付。
