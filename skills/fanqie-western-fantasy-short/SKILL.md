# 番茄西幻 / 哥伦布计划短篇写作工具

本工具继承 `skills/fanqie-novel-template/SKILL.md`，只补“西方经典设定 + 番茄短篇强情绪爽点”的题材专精方法论。通用的字数、目录、三件套、逐章写作、审核流水线和 AI 味 5% 闸门，以 `fanqie-novel-template` 为准，不在本 skill 里重复。

核心原则：**西幻设定只做情绪放大器，不做百科设定集**。短篇读者追的是开篇羞辱/献祭/拒绝/契约/死亡后的觉醒、反击和仪式场翻盘，不是复杂族谱、魔法史或神学解释。

## 审核优化后基线

`审核优化使用/人工样本共通点与审核校准报告.md` 显示，8 个高质量人工类型文正向样本当前校准后 AIGC 均低于 5%，其中多数落在 4.80% 封顶附近，最低约 2.14%。这批样本不是靠“乱写”过审，而是靠稳定的类型文控制力。后续写哥伦布计划短篇时，优先复制这些可验证的人味结构：

- **场景先行**：先落圆桌、座钟、面具、雨、泥、门、灯、血迹、纸笔、票据、合同、戒指、请柬、徽章、设备提示，再让情绪从现场后果里露出来。
- **动作链密集**：人物会看、摸、退、问、递、擦、站、环视、停顿、藏东西、改签名、按下按钮；不要只写“她很愤怒/他终于明白”。
- **对话轮次高**：关键信息通过问答、追问、打断、反驳、错认、沉默后补一句展开，不靠旁白一次性讲完。
- **疏密自然**：人工样本段长 CV 约 0.57-0.76，句长 CV 约 0.61-0.83；允许短句、半句、动作单行和中长句交替。不要每段都 80-120 字整齐铺开。
- **类型名词可重复，但功能必须变化**：血契、月银、欠费单、舞会请柬、实验编号等可以反复出现，但每次要承担新功能：规则揭示、证据、交易、威胁、选择或反转。
- **内容完整性干净**：绝不使用 OCR 脏码、随机汉字串、稀有神怪名词堆叠、拟声长串、无信息清单等绕检噪声；这类风险会触发内容完整性硬下限。

已验证的短篇审核经验也说明：低套路密度、私人毛刺、反爽结尾、段落重复清零会明显降低 AI 味；最容易留下 AI 指纹的是“中心反转太工整、金句化收束、情绪被讲穿、首尾闭环过干净”。本 skill 生成正文时，必须把“可读场景成立”放在“检测扰动”之前。

### 2026-07 反模式修正：不要把“人味细节”写成生成器痕迹

上一轮哥伦布批量稿暴露出一个严重问题：脚本门控通过，不等于读者读感通过。尤其要禁止下面两类写法：

1. **段首词牌化**：每段都以“地点/物件词 + 逗号”开头，例如“投屏餐桌，”“体育馆更衣室外的风……”。这种写法看似有场景锚点，实际像给段落批量贴标签。
2. **句尾尾巴化**：每句末尾都补一个“物件 + 动作 + 数字”的短尾，例如“名单多出1格”“柜门响2下”“银光偏3寸”。如果这个细节不能改变人物选择、暴露规则、形成证据或制造后果，就不是人味，是检测扰动。
3. **变量拼接化**：把“场景别称 / 标题别称 / 证据别称”硬塞进模板，形成“灰岩继任礼被人挡住的角落的狼席”“这场事这件事”“眼前这笔账这页”“记录员梅芙刚停笔的地方多晒一会儿”这类语法叠床架屋。此类文本即使有具体物，也按高 AI 味处理。
4. **视角混用**：同一自然段里既写“薇拉拿起……”，又写“我把……”，除非第二个“我”在直接引语内。正文必须先决定第一人称或第三人称，不能靠模板混用。
5. **世界观串线**：狼人、吸血鬼、女巫等西幻古典场景里不能为了“具体”乱塞“截图、发票、投屏、直播、邮箱、订单栏”；美高/学院可以有现代媒介，但必须来自场景逻辑，例如手机录像、校报邮箱、投屏、订单页。
6. **语义拼接化**：动作和物件必须互相能成立。禁止“用九级台阶把它摆正”“薇拉撕住月银戒指”“长桌边旁”“协议签名多晒一会儿”“清点页写得很乱：A、B、C……”这种看似具体、实际没有可表演动作的句子。

哥伦布计划正文宁可少一个“私有细节”，也不能把细节当装饰条。私有细节必须长在戏剧单元里：角色先看见/摸到/听到某物，随后因此误判、追问、退让、亮牌、失去入口或拿到证据。只要删掉这个细节不影响后续剧情，它就不该出现。

正文生成时必须优先通过“读者读感门”：

- 抽读任意 5 个自然段，段首不能呈现同一格式；若超过 30% 段落以“名词性场景词 + 逗号”开头，本章退回重写。
- 抽读任意 20 个句子，不能连续出现“句子主体 + 逗号 + 微动作/数字尾巴”的同构句；若出现 3 次以上，本章退回重写。
- 抽读任意 10 个“具体细节”，必须能回答“谁因此改了动作 / 谁因此暴露破绽 / 谁因此失去位置”。答不上来就删，不得留作检测扰动。
- 抽读任意 10 个长句，不能出现“X 的 Y 的 Z”“这场事这件事”“这笔账这页”“最亮的位置的……”等变量拼接痕迹。命中 1 次，本章退回重写。
- 检查全文视角：叙述主体是第一人称就不要写“薇拉拿起”；叙述主体是第三人称就不要在旁白里写“我把”。引号内自称不算。
- 检查世界观词库：每章列出 5 个最突出的物件，必须属于本赛道。狼人章应优先是戒指、狼席、席牌、药碗、验戒石；吸血鬼章是血契、蜡封、银杯、名册、棺室钥匙；女巫章是账本、收据、药瓶、铜秤、传票；美高章才使用手机、投屏、底片、校报邮箱。
- 检查动作可表演性：每个动词必须能被真人在现场做出来。“拿/递/压/扣/退/翻”后面接具体物，“九级台阶/第一行/证据/这页账”不能当工具使用。
- 数字只在剧情需要精确时使用：座位号、契约条款、账本页、订单金额、倒计时、编号。禁止为了区分句子写“第几格、几线、几处、几下”。
- 不允许用统一函数、统一前缀、统一句尾模板批量装饰段落。生成器、子代理或人工写作都必须按“动作-信息变化-人物反应”自然断段。

## 赛道选择

每本短篇最多选 **1 条主赛道 + 1 条副钩子**。不要把狼人、吸血鬼、女巫、魔法学院、赛博全部塞进一本 2 万字短篇。

### 1. 狼人 / Alpha / Luna / Rejected Mate

高识别度最高，适合番茄强情绪开局。

可用第一钩子：
- 月圆继任礼上，Alpha 当众拒绝命定伴侣。
- 女主被迫签拒绝伴侣协议，继妹顶替她成为准 Luna。
- 女主被羞辱性地送给北境战争 Alpha，结果错嫁登王座。
- 女主以为自己是狼群弃女，实则是被偷走的狼人公主。

短篇化公式：

```text
公开拒绝/逐出狼群 → 女主切断链接不求复合 → 边境/王族/死敌阵营接住她 → 女主觉醒 Luna/王族能力 → 原狼群反噬崩盘 → 月圆议会/加冕礼审判旧人
```

### 2. 吸血鬼 / 血契 / 新娘 / 献祭制度

适合替嫁、先婚后爱、血包误判、追妻火葬场。

- 女主替姐姐嫁给吸血鬼王子，婚礼当天被称为贡品。


## 人设与关系动力

### 女主模板

1. **被拒绝的命定者**：开篇被 Alpha/未婚夫/学院/家族公开否定，但她不求复合，第一反应是切断关系、保留证据、寻找新阵营。
2. **被献祭的交易者**：看似贡品、血包、新娘，实际有明确目的：换视力、查名单、救姐姐、夺继承权。
3. **被污名的女巫**：别人说她灾厄、廉价、邪恶，她用魔药、账本、预言或审判权证明污名源头是权贵。
4. **废柴商业脑**：无魔力、欠费、天赋归零，但懂交易、代购、供应链、证据链，用现实逻辑打穿魔法秩序。
5. **美高异类 / 寄住者**：被上流圈当装饰品或玩具，反手掌握社交秘密、合同条款和家族丑闻。

| 男主突然无条件宠 | 加一条制度束缚或利益冲突，让他必须公开选边。 |
| 仪式场翻盘像 PPT | 写清场地、观众、证物、规则触发、沉默 3 秒后的第一个反应。 |
| 西方背景像换皮都市 | 每章至少有 1 个不可替换的西方场景物件：月银、古堡钟、舞会请柬、法师塔徽章、校董席牌、血契蜡封。 |
| 情绪讲穿 | 删除“终于明白/复杂情绪/这意味着/从此以后”类解释，让动作、对话和现场后果承载。 |
| 中心意象过度漂亮 | 核心意象最多 1 个；不要“戒指-王冠-月光-玫瑰”全套闭环。保留一个生活毛刺打破精致感。 |
| 章尾金句化 | 章尾优先用证物变化、动作停顿、突发通知、人物没说完的话，不用排比和格言。 |
| 人味化后处理痕迹 | 不要为了检测插入无关短句、随机拟声或条款清单；只增加场景有效内容。 |
| 640 字分片局部过稳 | 在同一场景内增加真实打断：配角插话、物件故障、手部动作、错误称呼、数字/编号变化。 |
| 段首词牌化 | 禁止每段都用“场景名/物件名 + 逗号”开头。改成动作或视线进入：谁推门、谁停笔、谁抢遥控、哪份证据先被看见。 |
| 句尾尾巴化 | 禁止每句末尾都贴“物件 + 动作 + 数字”做差异化。需要细节时，把它写进因果链：它让谁误判、谁追问、谁改规则、谁失去入口。 |
| 编号滥用 | “第几格、几线、几处、几下”只有能被角色读、签、查、对账、追责时才保留；不能作为去重标记或节奏填充。 |
| 变量拼接化 | 发现“这场事这件事”“眼前这笔账这页”“X 的 Y 的 Z”或“被人挡住的角落的狼席”这类模板叠词，整章退回。重写时只保留一个主语、一个场景、一个动作，不再套别称。 |
| 视角混用 | 旁白不能同段出现“薇拉拿起……”和“我把……”。第一人称就全章用“我”，第三人称就全章用角色名/她；直接引语内自称除外。 |
| 世界观串线 | 不同赛道物件不能混用。狼人文不写截图/发票/投屏，吸血鬼文不写订单栏，女巫文不写直播控制台；现代媒介只在美高/现代学院逻辑内使用。 |
| 语义拼接化 | 删掉“动作 + 随机物件/位置/编号”的句子，改成可拍出来的动作：戒指可以戴上、退下、按在协议上；台阶只能走、停、绊倒，不能用来摆正东西。 |
| 脚本通过但读感像生成器 | 直接判未过。先抽读 5 段和 20 句，若段首/句尾同构明显，回正文重写，不得用 AIGC 低分覆盖。 |
| 细节平均铺满 | 删掉过场里的装饰细节，把笔墨集中到羞辱、追问、反击、证据暴露和代价兑现处。短篇需要疏密，不需要每句都“有物件”。 |

终版必须满足：
- 每章都有 `reviews/NN_ai_gate.json` 与统一审核报告 `reviews/NN.md`，机械门禁未通过时不得合并和交付。
- `审核报告_<书名>.md` 明确 AI 创作度不高于 5%。
- 段落重复、长句重复清零。
- 每章至少 2 个不可替换的私有细节：契约编号、账本页码、家族纹章、舞会座位、药剂名称、实验编号、校董姓名、旧日记日期等。
- 不得出现写作工程词泄漏，如“爽点推进、伏笔回收、关系升级、情绪爆发、AI 味、降重、模板”等。
- `aigc_value.py` 如出现分片风险下限、内容完整性、脏码或真重复提示，必须先修正文，不得靠整体均值过关。
- 人工读感门也必须过：任意抽样段落不得呈现统一前缀、统一尾句、数字尾巴、物件清单或“为了通过检测而塞细节”的痕迹。

## 一页 SOP

1. **确认赛道**：从狼人、吸血鬼、女巫、魔法学院、美高财阀、赛博人外中选 1 主 1 副。
2. **写高概念标题**：标题必须包含开篇冲突或终局反转。
3. **建故事圣经**：使用通用故事圣经模板，补上“西幻规则、仪式场、核心物件、旧秩序、新秩序”。
4. **定一条规则**：全书只保留最关键的怪物/制度规则，并规定它如何反噬反派。
5. **定人味锚点**：为全书列 12-20 个私有物件/数字/称谓/错误信息，每章至少使用 2 个，且功能变化。
6. **排 8-10 章细纲**：每章写清“本章爽点 + 证据/物件 + 对话冲突 + 章末钩子”，避免每章同构。
7. **写第一章前三行**：必须出现拒绝、献祭、羞辱、危机、契约或身份落差。
8. **逐章写作**：一次只写一章；按戏剧单元自然断段，不使用统一段首前缀、统一句尾尾巴或数字去重痕迹；每章完稿后数本章字符数、生成本章 `reviews/` 机械审核、跑本章人工化执行清单、审核、回写故事圣经索引和相关设计状态。
9. **中段检查**：第 4-5 章必须已有一次小翻盘，不能还在铺背景；第 6 章必须出现反派换规则或证物受损。
10. **结尾仪式场**：月圆议会、峰会、舞会、审判、学院大赛等只选一个，所有核心伏笔在这里回收，但保留一个现实代价。
11. **完稿质检**：按 `fanqie-novel-template` 执行字数核验、逐章 `reviews/` 机械门禁、整本降 AI 味、错别字、`aigc_value.py`、`text_signals.py`、`paragraph_dup.py`、`content_lint.py`、审核报告；额外执行人工读感门，抽读段首、句尾和数字细节是否有生成器痕迹。
12. **按报告回改**：优先修真重复、工程词、脏码、情绪讲穿、金句化章尾、场景锚点不足，再重跑脚本。
13. **生成交付物**：单章文件、故事圣经、`reviews/`、审核报告、可选合并稿、图片生成方案。
14. **最终回复**：说明目录、章数、字符数、AI 味闸门是否通过；未通过则继续改，不得交付。
