# 小说正文分段规范

来源摘要：
- Helping Writers Become Authors: 小说分段像“转向灯”，不只表示新想法，也帮助读者判断谁在行动、谁在说话，并调节动作和节奏。
- Gotham Writers Workshop: 对话交换中通常换说话人就新起段；叙事段则在焦点、想法、方向、时间或地点发生转移时换段。
- Jami Gold: 绝大多数情况下，每个说话人的对白应在自己的段落；动作与反应是否同段，要看焦点是否仍在同一角色。
- Richelle Braswell: 短段能加速或强调，但过多会变碎；长段能放慢，但过长会让读者感到文字墙，长对白也容易变成独白或信息倾倒。
- 故事革命: 长对白或长引述不应一直塞在同一个引号段里；对白要有角色语气和词汇特征，让读者可凭声口辨认说话人。

## 1. 分段的核心目的

小说正文的分段不是论文式“一个中心句一段”，也不是为通过检测随便切行。它服务四件事：

1. 让读者知道焦点换了：谁在看、谁在说、谁在做、谁在承受反应。
2. 控制阅读速度：短段用于冲击、打断、转折；长段用于观察、压迫、内心整合。
3. 给多角色场景定位：换说话人、换行动主体、换权力位置时，让读者不迷路。
4. 避免文字墙和碎片网格：段落有疏密变化，但每段必须有页面功能。

## 2. 什么时候必须换段

- 换说话人：新角色开口，原则上新起段。
- 换行动主体：上一段写 A 的动作/反应，下一步转到 B 的行动或判断。
- 换焦点：从屏幕字段转到人物反应，从会议桌转到走廊，从证据转到私人记忆。
- 换时间或地点：即使只过了几分钟，只要场景位置或时间感改变，通常新段。
- 新信息落地：证据、门禁、邮件、电话、封存状态等改变局面时，可单独起段或成为新段开头。
- 情绪反应需要停顿：重要反应可以新段，但不要连续用“孤句段”制造假紧张。

## 3. 什么时候不该换段

- 同一角色说话和紧贴的动作 beat 可放同段，尤其动作解释这句台词的动机。
- 同一观察链未换焦点时，不要为了显得“网文”硬拆成三四段。
- 旁观者同时惊呼、群体反应不需要逐人分段，除非某个人的反应改变局面。
- 纯装饰性动作不能单独成段：抬眼、停笔、看屏幕、顿了一下这类动作要合并或删除。

## 4. 段长软阈值

中文长篇/女频/悬疑职场的移动端页面可用以下默认预算：

- 20 字以内：孤句强调。每章少量使用，不能连续。
- 30-90 字：短反应、单轮对白、证据落点、动作转折。
- 90-150 字：常规叙事段，适合动作 + 反应 + 一句对白。
- 150-220 字：慢速观察、复杂心理、多人场面整合；使用前检查是否混入多说话人。
- 220 字以上：默认视为文字墙候选。只有长内心、长环境压迫或单人长述有充分理由时保留。

这些不是硬字数合同。真正的判断标准是：一个段落是否只承载一个清晰焦点；读者是否能一眼知道行动主体；读完后局面是否发生变化。

## 5. 多角色对话分段

推荐形态：

```
傅行简合上确认稿。“纪要写设备校验异常，明细不用入。”

门边的程棠把电脑又掀开一条缝。“设备校验能带出请假审批？”

傅行简没看他。“现在不做会后复盘。”
```

不要把三个人的台词塞在同一段里，除非目的是制造“同时嘈杂”的群体效果，且读者不需要逐个识别说话人。

## 系统消息

以 `【...】` 呈现的系统私聊、任务、规则和结算，一条消息独立一个自然段。人物问句在上一段，系统回答在下一段；人物读完后的反应再另起叙述段。两条系统消息也不能写成 `【A。】【B。】` 粘在一起，应删并、分层，或让人物在中间真实回应。

独立成段不等于连续碎屏。一次交互优先保留一问一答，正式数据行和陪伴吐槽分开，系统连续发言超过三条时优先压缩。

## 6. 第一章类场景的适用规则

数据室、会议、审计、权限这类场景容易写成大段流程说明。落页时按以下方式切：

- 屏幕/系统状态单独承担新信息。
- 角色反应跟随该角色，换人就换段。
- 管理口径、外审边界、安全组甩锅分别给段，不塞进一个“会议纪要段”。
- 家庭线电话进入时换段，因为它改变压力来源。
- 章末必须以“下一步动作/空项/倒计时/证据缺口”收住，而不是长段抒情。

## 7. 交付自检

- 是否有 220 字以上段落？如果有，说明它为什么不能拆。
- 是否有一段里三个人轮流说话？若不是群体噪声，拆。
- 是否连续 3 段长度接近、功能接近？换一个段落入口：对白、物件、环境或反应。
- 是否短段太多？把纯停顿、纯动作、纯情绪并回相邻段。
- 是否长段太多？优先按换说话人、换焦点、换证据落点拆。
