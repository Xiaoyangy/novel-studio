# 网络参考与热梗写作边界

本文件规定网络检索资料、最新生活细节和热梗如何进入小说写作链路。它们只能增强真实感、时代纹理和场景材料，不能替代本书大纲、角色状态、世界规则、资源账本、用户规则或审核结论。

## 默认检索原则

1. 默认使用最新资料。除非用户指定历史年代，否则优先检索当前年份、最近 30-90 天或平台仍在流通的资料。
2. 记录检索时间、查询词、来源链接、来源类型和可用结论。没有记录的网络资料不能写成“已核实事实”。
3. 官方规则、行业流程、价格、平台政策、法律法规和新闻事实必须以可信来源为准；娱乐梗、流行语和社交口头禅可以参考公开盘点，但只当语言材料。
4. 网络资料进入正文前必须转化为本书可见细节：物件、屏幕、账单、通知、队列、群聊、口头误解、生活麻烦或角色选择。不要把网页摘要搬进旁白。
5. 外部资料与本书已建事实冲突时，服从本书事实；资料只作为现实支架或类比，不反向改写世界观。
6. 默认排除涉政、灾难、社会冲突、刑案、公共安全事故和其他敏感事件；不要为了“热点感”蹭真实敏感事件。需要社会背景时，转成虚构、低敏、生活化的场景压力。

## `web_reference_brief`

每本书在正文写作前必须提供项目级网络参考简报，或在当轮 `external_reference_plan` 中记录等价的 web search 证据。简报位置：

- `meta/web_reference_brief.md`
- 或 `meta/web_reference_brief.json`
- 或 `references/web_reference_brief.md`

简报应包含：

- `retrieved_at`：检索日期和时区。
- `freshness_policy`：为什么这些资料仍可用，什么时候需要刷新。
- `queries`：主要查询词。
- `safety_filter`：排除的涉政/敏感/灾难/冲突类话题，以及替代的低敏生活语境。
- `sources`：来源链接、来源类型、可信度备注。
- `usable_details`：可以转化为正文的生活细节、行业流程、物件、界面、口语样本。
- `trend_language`：热梗/流行语的含义、适用人群、可承载角色、使用预算。
- `excluded_items`：不适合本书、不可信、过时、太出戏或版权风险高的内容。
- `transformation_rules`：每条资料如何从网络语言转成场景动作、对白摩擦、屏幕信息或物件证据。

`novel_context` 会把简报放入 `reference_pack.references.web_reference_brief`。Writer 必须在 `causal_simulation.external_reference_plan` 和 `trend_language_plan` 中说明本章使用了哪些资料、如何使用、哪些不用，以及长梗如何后续回收。没有网络简报或当轮检索证据时，不允许进入正文写作。

## 热梗使用规则

热梗不是装饰词库。它只适合以下场景：

- 配角、群体角色、弹幕/短视频外放、小区群、同事群、外卖/物业/客服等当代生活载体。
- 用来显示人物年龄、平台习惯、职业环境、社交误判或现场嘈杂，不用来解释设定。
- 每章默认 0-2 处明确流行语；更多只能作为弱化后的口语节奏或误听，不写成梗串。
- 主角内心旁白、关键恐怖叙述、规则条款、章末钩子默认不用热梗，除非它本身就是世界规则的讽刺对象。
- 不能让角色突然说不符合年龄、职业、压力和关系的热门词；也不能把过时梗当“年轻感”。

优先转化方式：

- 原梗变为角色半句口头反应，例如“这也能算钱？”而不是完整网络金句。
- 热词变为现实物件或界面，例如短视频外放标题、群聊昵称、平台弹窗、物业通知。
- 用“听不全、说岔、被打断、别人不接话”制造现场感，而不是让台词整齐押梗。
- 梗只服务冲突：让人物误判、暴露恐惧、拖慢确认、造成关系摩擦或提供生活证据。

## AI 味风险

网络资料会放大三类 AI 味：

1. 资料堆砌：连续列词、列平台、列政策、列流行语。
2. 旁白百科：角色不行动，作者替资料做总结。
3. 硬贴时代感：恐怖、悬疑或古风场景里突然出现不合角色的流行词。

写作前必须在 `anti_ai_execution_plan` 中为外部资料写清阻断策略：句式不整齐、资料不连列、至少一处误判/动作后果、热梗有载体、有预算、有退出。

## 审核标准

Editor 检查网络参考时，重点不是资料数量，而是：

- 是否有 `web_reference_brief` 或 `external_reference_plan` 说明来源和时效。
- 是否避开涉政、灾难、社会冲突、刑案、公共安全事故和其他敏感事件。
- 正文是否把资料转成可见场景，而不是解释段。
- 热梗是否符合角色、关系和压力；删掉热梗后，场景功能是否仍成立。
- 是否出现过时、低可信、错时代、错平台或侵犯版权的表达。
- 是否因为追求“新”破坏本书语气、人物声口和章节契约。
